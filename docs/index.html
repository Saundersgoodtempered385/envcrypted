Commands

Four commands.
Complete control.

Everything you need to secure, share, and sync your environment variables.

envcrypted init Setup

Initialize envcrypted in your project. Generates a cryptographically secure master key (or accept your own) and sets your preferred storage type — local vault or GitHub auto-push.

$envcrypted init
envcrypted push Encrypt

Encrypts your .env file using AES-256-GCM and saves it as .env.vault. If GitHub storage is selected, it automatically commits and pushes the vault to your repository.

$envcrypted push --message "update keys"
envcrypted pull Decrypt

Pulls the latest vault from GitHub (if applicable) and decrypts it back into a .env file using your master key. Perfect for new team members getting started instantly.

$envcrypted pull
envcrypted audit Security

Scans your .env for critical security issues — weak passwords, exposed DB URIs with credentials, placeholder keys, localhost URLs in production, debug flags, and more.

$envcrypted audit
Security

Bank-grade encryption.
Zero compromise.

Built on Node.js's battle-tested crypto module. Open source. Auditable by anyone.

Algorithm AES-256-GCM
Key Derivation PBKDF2-SHA512
Iterations 100,000
Salt 64 bytes (random per encryption)
IV 16 bytes (random per encryption)
Auth Tag 16 bytes (tamper detection)
Master Key Storage Never stored anywhere
Open Source MIT Licensed

🔑 How the master key works

1
You provide a master key (generated or custom)
↓
2
PBKDF2 derives a 32-byte encryption key from it (100K iterations)
↓
3
AES-256-GCM encrypts your .env with a random salt + IV
↓
4
Encrypted vault is safe to commit. Master key is never stored.
↓
5
Without the master key, the vault is computationally impossible to crack
